🚀 FER CMS 2026 — Plan izgradnje

Reusable "builder plate" (starter kit) za nove projekte — isti modularni model kao stari CMS (fer/* obitelj paketa), ali na modernom Filament stacku. Svaki paket se verzionira, updatea i razvija zasebno.

PHP 8.4+ Laravel 13+ Livewire 4.3+ Filament 5.6+ PostgreSQL 16+ · pgvector AI: Claude via Prism
✅ native — Filament/Laravel 🧩 3rd-party plugin — wrappamo ga 🔧 naš kod — fer/* paket 💰 plaćeni plugin

1Koncept: "builder plate"

Starter kit iz kojeg se novi klijentski projekt digne za par sati; zajednička logika živi u verzioniranim fer/* paketima. Ključni idiom: svaki fer/* paket = Filament v5 plugin (iznimka: fer/support — čista biblioteka bez UI-a).

PROJEKT (klijent)

composer create-project fer/cms-starter → skeleton · samo: entiteti, blokovi specifični za klijenta, tema/dizajn

require — fer/cms (metapaket) povuče sve fer/* pakete

fer/* PAKETI — Filament v5 plugini, zasebno verzionirani

supportpanelsettings pagesblocksmenus mediai18narticles formsseosearchai
wrap / extend

FILAMENT 5 + community pluginovi

Spatie · awcodes · Lara Zeus · Ralph J. Smit · TomatoPHP …

LARAVEL 13 · Livewire 4 · PHP 8.4 · PostgreSQL + pgvector

Kako se paketi uključe u projekt (Filament plugin kompozicija — nasljednik starog "registriraj resurse/menije"):

// app/Providers/Filament/AdminPanelProvider.php
$panel
    ->plugin(FerPanelPlugin::make())      // branding, dashboard, RBAC
    ->plugin(FerSettingsPlugin::make())   // postavke klijenta (info, kontakti, socials)
    ->plugin(FerPagesPlugin::make())      // stranice + content builder
    ->plugin(FerBlocksPlugin::make())     // paleta blokova
    ->plugin(FerMenusPlugin::make())      // menu builder
    ->plugin(FerMediaPlugin::make())      // media biblioteka
    ->plugin(FerI18nPlugin::make())       // prijevodi + jezici
    ->plugin(FerArticlesPlugin::make())   // objave/blog modul
    ->plugin(FerFormsPlugin::make())      // kontakt forme + upiti
    ->plugin(FerSeoPlugin::make())        // SEO + AEO/GEO
    ->plugin(FerSearchPlugin::make())     // search
    ->plugin(FerAiPlugin::make());        // AI (prijevodi/pisanje/asistent)

Uključuju se samo paketi koje projekt treba.

2In-house fer/* paketi

Za svaki: Uloga · Wrappamo (3rd-party) · Pruža (+ napomena gdje je bitna). Podijeljeni u tierove.

🧱Tier 1 — Temelj

fer/supportshared library (no UI) · nasljednik fer/contracts
Uloga
Zajednički temelj: contracts/interfaces, enumi, traitovi, helperi, base config, base ServiceProvider. Sve ostalo ovisi o njemu.
Wrappamo
spatie/laravel-package-tools — standardni boilerplate za gradnju Laravel paketa
Pruža
HasFerTranslations, FerModel base, locale registry contract, config publishing, shared traits/enum.
Napomena
Thin package — samo shared kod (interfaces, traits, helpers), bez UI-a u panelu. Drži konvencije koje svi paketi dijele.
fer/panelnasljednik fer/admin jezgre
Uloga
Bootstrap Filament panela: branding (FER logo/tema), navigacijska struktura, dashboard, RBAC — ali tanak (Filament radi teški dio).
Wrappamo
✅ Filament 5 Panel + 🧩 bezhansalleh/filament-shield (RBAC na spatie/laravel-permission)
Pruža
Gotov panel s FER izgledom, standardne navigacijske grupe, role/permission setup, MFA (native u Filamentu), base Resource klase, Blueprint konfiguracija za scaffolding.
Napomena
Ovdje živi zajednički look & feel svih FER projekata.
fer/settingsnasljednik ProjectSettings koncepta
Uloga
Postavke projekta/klijenta na jednom mjestu — sve što se pojavljuje u footeru, headeru, kontaktu i schema.org: naziv tvrtke, adresa, OIB/VAT, telefoni, emailovi, social links, radno vrijeme, Google Maps, analytics/tracking ID-jevi, default OG slika.
Wrappamo
spatie/laravel-settings (typed settings klase u bazi) + službeni Filament settings plugin (gotove "Settings" stranice u panelu)
Pruža
Settings grupe (Tvrtka / Kontakt / Social / Tracking…) kao tabovi u panelu, translatable polja gdje treba, Blade/helper API za frontend — npr. settings('company.address').
Napomena
Footer, header, kontakt stranica i Organization JSON-LD u fer/seo čitaju odavde → klijent sam ažurira podatke, bez developera.

📄Tier 2 — Sadržaj

fer/pagesnasljednik Site/Tree/StreamField dijela
Uloga
Stranice: hijerarhijsko stablo (nesting/children), content Builder, tipovi stranica → template, frontend routing/render.
Wrappamo
kalnoy/nestedset (nested-set hijerarhija u bazi) + solution-forest/filament-tree (drag&drop stablo u panelu) + ✅ Filament Builder field
Pruža
Page model + stablo + drag&drop reorder, content Builder polje, page-type→template resolver, frontend rendering blokova, publishing statusi (draft / published / scheduled), preview, content revisions.
Napomena
Page-modul gradimo sami — postoji gotov plugin "Fabricator" (block-based page-builder skeleton), ali se loše slaže s višejezičnošću i našim publishing workflowom, pa ga ne koristimo. Paleta blokova dolazi iz fer/blocks.
fer/blocksreusable block katalog · nasljednik StreamFieldsService palete
Uloga
Reusable katalog content blokova + njihove frontend render komponente.
Wrappamo
✅ Filament Builder::block() + Repeater + native RichEditor (TipTap: custom blocks, merge tags, HTML/JSON pohrana)
Pruža
  • Blokovi: text/section, heading, quote, image, image+alt, gallery, slider, embed/video, CTA, FAQ, documents, pricelist… — svaki s Filament schema() + Blade render komponentom
  • Relacijski blokovi: Select/CheckboxList polja unutar bloka vezana na modele (BelongsTo/BelongsToMany + pivot podaci) — blok sprema ID-jeve, render resolva relacije pri prikazu
Napomena
Zaseban paket da katalog verzioniramo neovisno; projekt dodaje svoje blokove preko fer/blocks API-ja. RichEditor custom blocks (drag&drop unutar teksta) komplementarni su Builderu — mali umetci (CTA, citat, slika) u rich textu.
fer/menusnasljednik fer/menu
Uloga
Menu builder: više menija (header/footer/…), stavke tipa page/custom/URL, drag&drop.
Wrappamo
datlechin/filament-menu-builder — drag&drop menu builder, više lokacija
Pruža
Menu resource, veza stavki na fer/pages modele, frontend menu render helper. Stranica koja nije ni u jednom meniju prirodno ispada iz navigacije ("nepovezano").
fer/medianasljednik fer/files
Uloga
Media biblioteka + konvencije: kolekcije, konverzije (WebP/AVIF), responsive, focus crop, fallbacks, translatable alt/SEO na mediju.
Wrappamo
  • spatie/laravel-medialibrary — temelj: originali, konverzije, kolekcije
  • tomatophp/filament-media-manager — centralna "Mediateka" na Spatie temelju
  • službeni filament/spatie-laravel-media-library-plugin — upload polja na resursima
  • spatie/laravel-image-optimizer + ✅ Filament FileUpload->imageEditor()
Pruža
  • Media model s translatable alt/title/caption, obavezan alt (a11y)
  • Konverzije: izlaz WebP/AVIF/PNG/JPEG · image editor: crop, rotate, flip, focal crop
  • Centralna "Mediateka": folderi/subfolderi, browser, picker polje za odabir iz biblioteke, drag&drop reorder, live preview
  • Multi-field uploadi preko named kolekcija · per-locale datoteke (npr. brošura HR/EN — locale-aware kolekcije 🔧)
Napomena
💰 Premium opcija: Media Library Pro (ralphjsmit/laravel-filament-media-library, ~€99) — najpoliraniji UI, bulk-upload, permissions, isto na Spatie temelju; isprobati obje pa odlučiti. Curator otpada — nekompatibilan sa Spatie (vlastiti Media model).
fer/i18nnasljednik fer/translator
Uloga
Prijevodi + upravljanje jezicima — "easy lang add": dodaj locale → sve dobije jezik, bez migracija.
Wrappamo
  • spatie/laravel-translatable — prijevodi kao JSON po atributu
  • lara-zeus/spatie-translatable — Filament language switcher (službeni plugin je napušten, Lara Zeus održava)
  • spatie/laravel-sluggable — per-locale slugovi · mcamara/laravel-localization — /hr/ /en/ rute
  • UI keys: spatie/laravel-translation-loader (stringovi u bazi) + kenepa/translation-manager (Filament sučelje)
Pruža
Locale registry, language switcher na formama/tablicama, per-locale slugovi, lokalizirano rutiranje, UI-keys prijevodi, "needs-review" flag. Novi locale se propagira i na admin panel — jedan config unos aktivira jezik u adminu, modelima, rutama i UI keys prijevodima.
Napomena
Dodavanje jezika = config unos (bez migracija). AI auto-prijevod dolazi iz fer/ai.
fer/articlesdefault modul
Uloga
Objave/blog modul — default sadržajni entitet svakog projekta (novosti, blog, aktualnosti).
Wrappamo
✅ Filament Resource + naši paketi (blocks, media, i18n, seo)
Pruža
Article model (kategorije, autori, published_at, featured image, Builder sadržaj), arhiva/kategorija/single frontend, RSS feed.
Napomena
Referentna implementacija "kako se entitet gradi na fer/* stacku" — ujedno predložak za custom entitete po projektu.
fer/formsdefault modul
Uloga
Kontakt forme + prikupljanje upita/leadova (s marketing podacima i privolama).
Wrappamo
✅ Filament (submissions inbox) · 🧩 opcionalno lara-zeus/bolt (full form builder kad klijent sam slaže forme)
Pruža
Frontend form komponente, submissions inbox u panelu (pregled, export), email notifikacije, honeypot/anti-spam, GDPR privole (marketing consent, retention).
Napomena
Default: par predefiniranih formi po projektu (pokriva 90% slučajeva); Bolt tek kad treba klijentski form builder.

🔎Tier 3 — Prezentacija & pronalaženje

fer/seoSEO + AEO/GEO za bilo koji model
Uloga
SEO + AEO/GEO (optimizacija za AI tražilice: ChatGPT, Perplexity, Google AI Overviews) za bilo koji model.
Wrappamo
ralphjsmit/laravel-seo + ralphjsmit/laravel-filament-seo (Filament polja) + spatie/schema-org + spatie/laravel-sitemap
Pruža
  • HasSeo trait (meta/OG/canonical po modelu), JSON-LD (Article/FAQPage/Organization), sitemap, hreflang
  • TOC, responsive tablice, lazy slike, llms.txt generacija
  • AEO polja: ai_summary, key_facts, faq
  • GEO polja: citation-ready summaries, entity/brand schema, quotable facts — strukturirano za AI search engines
  • Redirects manager (301 pri promjeni sluga — spatie/laravel-missing-page-redirector)
Napomena
AEO/GEO dio je 🔧 custom — nema gotovog paketa. AI generira ai_summary (iz fer/ai). Organization JSON-LD se auto-puni iz fer/settings.
fer/searchfull-text + semantička pretraga
Uloga
Pretraga: klasična full-text + semantička (višejezično).
Wrappamo
laravel/scout + Meilisearch/Typesense; PostgreSQL pgvector za semantic
Pruža
Searchable trait, hybrid search (keyword + semantic), Filament global-search integracija.
Napomena
Embeddings pipeline dijeli s fer/ai (pgvector kolone).

🤖Tier 4 — Inteligencija

fer/aiAI sloj
Uloga
Auto-prijevodi, pisanje sadržaja, blog generacija, RAG asistent, embeddings za semantic search.
Wrappamo
Prism (+ Laravel AI SDK) · Claude (Anthropic provider) · pgvector · embeddings (Voyage/OpenAI)
Napomena
Laravel AI SDK je javni sloj (agent klase, conversations, streaming); Prism je underlying engine dok SDK ne sazrije — API prema projektima ostaje AI SDK-kompatibilan. Ugraditi cost kontrole: sve preko queue-a + rate limit, cache odgovora, budget po projektu, log potrošnje tokena.
SposobnostKakoModel
Auto-prijevodi (UI keys + sadržaj)artisan komanda / queue job → batch → JSON izlaz; "needs-review" flagClaude Haiku klasa + Batch API (50% jeftinije)
Content writing (inline pomoć)Filament field actions ("✨ napiši / proširi / skrati / promijeni ton") pored poljaClaude Sonnet klasa
Blog generacijatema/outline → structured output → Filament Builder JSON draft (naslov, sekcije, FAQ, SEO, ai_summary)Sonnet/Opus klasa
RAG asistent (docs za klijenta)docs → embeddings → pgvector → Claude odgovara s citatima; Filament panel widget (Livewire streaming)Claude Opus klasa + embeddings
Semantic search (za fer/search)sadržaj → embeddings → pgvector cosine similarity → hybridembeddings (Voyage/OpenAI)
⚠️ Embeddings ≠ Claude. Za semantic search/RAG vektore treba embeddings model (Voyage voyage-3 ili OpenAI text-embedding-3). Claude radi generaciju/odgovore. Prism/AI SDK vode oba providera paralelno.

5Razvoj & održavanje paketa

TemaOdluka
Repo strategijaMonorepo (fer-projekt/cms) sa svim fer/* paketima u packages/* tijekom razvoja → path repositories za lokalni rad. Publish: subtree split (GitHub Action) u read-only pojedinačne repove.
DistribucijaPrivatni Composer repo — nasljednik starog packages.fer.plus → Private Packagist ili Satis. Projekti rade require "fer/pages": "^2.0" itd.
VerzioniranjeSemVer po paketu — breaking change u fer/blocks ne ruši fer/seo. Projekt pinna verzije i updatea kad želi (composer update fer/pages).
KvalitetaTestovi: Pest + Orchestra Testbench (testiranje paketa izvan aplikacije). CI po paketu: pint (lint), phpstan (static), testovi. Svaki paket: composer.json, README, CHANGELOG, config publish, migracije.
ScaffoldingFilament Blueprint + make:filament-resource → novi entitet u projektu koristi naše fer/* traitove (SEO, translatable, media). Zamjena starog code generatora.
